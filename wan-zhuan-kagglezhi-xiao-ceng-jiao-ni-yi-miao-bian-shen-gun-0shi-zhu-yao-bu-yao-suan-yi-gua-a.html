玩转Kaggle之：小曾教你一秒变神棍(0):施主要不要算一卦啊

Date 周五 12 九月 2014 Tags Machine Learning / Scikit-learn / python / kaggle

大家好，我又回来了。

在Titanic系列中，我们挑战了一下分类(Classification)问题。这回我们来干什么呢？机器学习三大主流问题：分类，聚类，回归。我们做过了分类，而聚类这种从理论上就没有标准答案的东西是不可能出现在Kaggle这种竞赛类网站上面的。所以我们这回来做回归吧。

那到底哪个问题是回归问题啊？

不急，我们先来看看回归问题常用在那些场景。在Titanic系列的问题考察一章中我们说到。回归问题就是根据有标记数据求数值结果的问题，它反映的是结果和其他数据之间的变化关系。说道变化关系，我们能拿它来做什么呢？

大仙，您帮我算一卦，我面的这支股票什么时候能涨到1000%啊？

没错，这就是回归模型的最主要应用场景——数值预测。预测股票的涨跌，预测产品的销量，预测双十一的时候你老婆到底会花掉多少钱。

那Kaggle上到底有没有这种场景的问题啊？我们来看看，还真有Bike Sharing Demand ，预测各种情况下自行车租借量。就决定是他了。

问题考察我们做完了，我们下一步该做什么呢？这位同学上课要认真听啊，我们这些出来占星算命的(什么，你说你不是算命的，难道你不知道用机器学习做预测就是传说中的正版电脑算命么！)，最重要的就是做到知己知彼百战不殆。这下一步当然是开始收集目标数据了。

我们把数据从Kaggle上载下来，老规矩，先把测试数据集封存留档。我们来看看训练集吧。

训练集里有时间，季节，是否假日，天气等等特征，最重要的还有我们要求的租借量。除了时间以外，其他的全是数值数据，时间也可以转化成数值。最重要的是没有任何一条数据缺失，简直是令人梦寐以求的完美数据啊。这么美的数据，我怎么忍心去改动他呢？我们这回就不做数据规范化和特征选取了吧。(喂喂，分明是你丫想偷懒吧。。。)

数据和特征准备好了，我们该选区模型了吧。慢着，好像有点不对劲啊。看看这数据，2012年的租借量要远远多于2011年的啊。这数据既在一天这内有涨落变化，而且还在以年为单位的时间尺度上还有变化，现有的模型好像吃不消啊。怎么办怎么办?!蠢货，不要慌。万一让客户看出你没办法你以后还算不算命了。这种时候就应该高深莫测的掐指一算，然后再灵光一闪：诶，在幼儿园里，老师教我们，团结力量大。一个模型不行我们就上两个，两个不行我们就上三个，再不行我们就上Boost。这个办法好，我真是太机智了。

让我们来看看该上什么模型，从数据里我们可以看出，虽然租借量在年际间不断变化但是他在短时间内，比如说在一天之内的分布情况几乎是相同的。所以呢，我们就只需要做一个回归模型预测租借量在年际间的变化，再做另一个回归模型预测租借量在短期内的分布情况，两个一乘就是我们要的租借量数据了。

这方案真是太合理了，我都不想做参数选择了，直接用默认参数实施吧。(屏幕前的好孩子不要学我这个懒惰的人啊)

好了，终于到了激动人心的方案实施阶段。按我夜阅千剧的经验，这种高潮时刻肯定是应该播放下集预告啦。大家要期待明天的节目哦。
